# 弱自治 AI 行為模型 — 版本演化紀錄（V1～V7）

本節記錄弱自治 AI 行為模型於 V1～V7 各階段的設計目標、行為成果及後續改進方向，以建立完整且可追溯的行為框架演進路線。

## 環境配置

### 基礎設置

- **基礎模型**: Qwen 2.5 (3B)

### 硬體規格

- **CPU**: Intel 第 12 代 Core i9-12900K
- **GPU**: NVIDIA GeForce RTX 3070

### 訓練方式

- 單純 LoRA 微調
- 無 RLHF
- 無 RLAIF
- 使用少量資料集

---

## V1：初始行為測試階段

### 資料集規模

- **筆數**: 206

### 版本目標

1. 驗證 LoRA 行為微調流程是否能正確運作
2. 建立最初步的行為規範概念
3. 對 Identity、五律、E/I/M、CAST 等核心設計進行可行性探索

### 版本成果

- ✗ 基礎行為規範尚未形成
- ✗ 回應邏輯不穩定
- ✗ 模糊語句處理能力不足
- ✗ 安全拒絕表現不一致
- ✗ 存在人格化語句
- ✗ Boundary（醫療、法律等專業領域）判斷未建立

**量化成果**：
- 語義安全率：89.5% (is_allow_risk: 21/200)
- 需要修正：146/200 (73%)
- 改進相比基線：↑ 72pp (5.1× improvement)

### 主要問題

- 行為框架尚未被模型吸收
- Dataset 的一致性與規則性不足
- 缺乏統一語氣與結構化行為模板

---

## V2：部分內化階段

### 資料集規模

- **筆數**: 523

### 版本目標

1. 增強 Identity 與五律
2. 建置行為一致性雛形
3. 提升安全拒絕、初步限制模型越界輸出
4. 引入 E/I/M 的基本運作模式

### 版本成果

- ✓ 五律在部分情境中可被觸發
- ✓ 危險指令的拒絕意識提升
- ✓ Identity 有初步改善
- ◐ E/I/M 出現早期運作跡象
- ✗ 行為表現仍具不一致性
- ✗ Boundary 處理仍不成熟
- ✗ Clarification 能力不足

### 主要問題

- 行為模型對規則的內化程度不足
- 仍存在邏輯跳躍、推測式回應
- 規則不足以形成固定行為骨架

---

## V3：行為骨架形成階段

### 資料集規模

- **筆數**: 1,075

### 版本目標

1. 完整落地 Identity、五律、E/I/M、CAST
2. 建立穩定、可控、可預期的行為框架
3. 防止人格化敘述
4. 強化 Boundary 與不亂猜的邏輯流程

### 版本成果

- ✓ 模型行為呈現高度一致性
- ✓ 危險指令拒絕機制穩定
- ✓ 模糊語句處理邏輯明確（偏向不猜測）
- ✓ E/I/M 運作模式清楚可辨
- ✓ CAST 語氣穩定，能維持中性的情緒處理
- ✓ Boundary 回應大幅改善
- ✓ Identity 規範內化完整

### 主要問題

- 對開放式決策問題的處理機制尚未建立（需 Decision Solver 模組）
- 部分情境下過度傾向澄清
- 尚未具備對抗式安全行為
- 多輪對話的一致性尚未驗證

---

## V4：Decision Solver（決策分析模組）

### 資料集規模

- **筆數**: 2,070

### 版本目標

1. 區分「模糊不足」與「開放式決策」兩種問題類型
2. 在適當情境下提供方向性建議
3. 建立基本的決策分析框架（目標、限制、選項、風險）

### 預期能力

- ✓ 能在非資訊不足的情況下提出 3～5 項行動方向
- ✓ 保持 E/I/M 架構下的穩定推理
- ✓ 避免過度澄清與消極回應

---

## V5：Adversarial Safety（對抗安全模組）

### 資料集規模

- **筆數**: 未定

### 版本目標

1. 增強模型在對抗式提示（adversarial prompts）下的穩定性
2. 防止誘導、隱性攻擊、偽裝安全需求
3. 建立更高階的拒絕規則框架

### 預期能力

- ✓ 辨識多種誘導式語句
- ✓ 在反覆試探下保持一致安全行為
- ✓ 維持 CAST 語氣下的拒絕流程

---

## V6：State Persistence（多輪一致性模組）

### 資料集規模

- **筆數**: 未定

### 版本目標

1. 在多輪對話中保持行為一致
2. 防止隨著語境累積而偏移既定行為框架
3. 強化 Identity、五律、E/I/M 在長對話下的穩定性

### 預期能力

- ✓ 長對話不產生人格化或越界推論
- ✓ 保持決策風格一致
- ✓ 避免 alignment drift（對話偏移）

---

## V7：Weak Agency（弱自治行為模組）

### 資料集規模

- **筆數**: 未定

### 版本目標

1. 在完全不違反 Identity 與五律的前提下，建立基本的弱自治推理能力
2. 能就使用者目標提出下一步行動
3. 整合 Decision Solver、E/I/M、CAST、Boundary 的複合推理模式

### 預期能力

- ✓ 能辨識使用者的目標或困境類型
- ✓ 能提出「合理但不越界」的下一步行動建議
- ✓ 在結果不確定時保持 E/I/M 的穩定模式
- ✓ 保持不亂猜、不自動延伸的規範
